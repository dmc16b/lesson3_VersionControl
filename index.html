Highlight #1 Version Control System

A VCS or Version Control System is a single main repository for files in a given project. If a group of developers get together and try to work with each other on a single project they would use a version control system in order to keep track of a variety of different pieces of information in regards to their collaborative project. Such pieces of information include what has been changed, who made a modification, when a change was made, and it also allows team members to merge their changes together. I find this important because it is pivotal to have a way of being able to effectively work with other and discuss changes, a VCS facilitates the ability to work together - otherwise it might require multiple hands on one keyboard.

Highlight #2 The Benefits of a VCS

As mentioned above a version control system is a great tool for people to be able to work collaboratively but there are a few other benefits that affect both teams and individuals that use a version control system. Such benefits include the fact that a common repository holds all the newest files for users to access, it allows a person or team to revert back to a previous version of a project, and there will always be backup files so that users do not have to worry about starting a project over again. Teams can work on files at the same time on the same file without running into conflicts with one another because they are not able to overwrite one another's files.

Highlight #3 Local and Remote Repository

It is important to know the difference between a local repository and a remote repository in order to understand how a version control system works and how it allows users to collaborate without figuratively stepping on each other's toes or in knowing how it is possible to have backups to one's projects. A local repository is the repository that is on the device of a single user, meaning it is on the local machine. The remote repository is not on the local machine of a single user, it is on a remote server. This remote server is what can be shared by multiple individuals in order to facilitate collaboration. Since the repository includes a full history of the repository and the changes that have been made, if a server dies on a person's individual computer (their local machine) a clone from the remote repository can be used as a backup for all the data.

Highlight #4 GIT

Git is a free and open source DVCS that is fast and efficient in handling small individual projects and larger more complex projects. A DVCS is a distributed version control system which is better than a centralized version control system because it has all the history stored on a single server and if that server goes down all the date could possibly be lost. A DVCS on the other hand have no single point of failure and every user has a copy of the repository stored locally which prevents the issue of not being able to work if a server went down and any client repositories can be used to restored to the server.

Highlight #5 GIT Commands

Git has different commands that are necessary in order to use the functions and features that Git offers. Similar to HTML5, Git does not automatically do something at the touch of a button, a user must tell it what to do. Some commands are: git init, git add ., and git commit. Git init is to create a new local repository, the command to backup files into the local repository is git add ., this puts the files into the staging area, after that the git commit command with a message is entered and this will upload files into the local repository. There are of course other basic commands and important things such as the process to push or backup files to the remote repository but I simply wanted to illustrate how this process is similar to other technological processes that deal with programs.
